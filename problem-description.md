# Ezra Full Stack Developer Take-Home Test

## Objective
Build a small to-do task management API and frontend. This test evaluates:
 - Backend API design. Please use .NET Core.
 - Data structure design. Please use SQL Lite or EF Core in memory.
 - Frontend component design. Please use React or Vue.
 - Communication between frontend and backend.
 - Clean code, architecture structure, and thought process.
 - Trade-offs and assumptions.
 - Include a short README.md with setup steps and your explanation notes.
 - Comments or a README.md explaining assumptions, scalability, and what you would implement in the future.
 - Submit a GitHub repo link with both frontend and backend projects.
 - Please add any features you feel are required for a Production MVP.

 ## Tips

Treat this like a small “production MVP”, not just a demo app. Think about what
would be needed if this were going to real users.

At the same time, we’re looking for thoughtful depth. Strong submissions go a bit
beyond the basics, for example handling edge cases, thinking through data
models, and considering how the system would evolve.

Include meaningful tests where appropriate. These don’t need to be exhaustive,
but should demonstrate how you think about correctness and reliability.

Be mindful of error handling and failure scenarios. Consider how your API
behaves in non-happy paths.

Pay attention to how you structure your API and data boundaries, and how the
frontend interacts with it.

Avoid unnecessary complexity. We’re not looking for an over-engineered
solution, but rather something simple, clean, and thoughtfully designed.

A good rule of thumb is to aim for something that would feel reasonable to ship
as a first version of a real product.
